# Search Console 索引治理设计

状态：已确认
日期：2026-07-15
对应需求：`docs/requirements/search-console-indexing/requirements.md`

## 设计原则

- 抓取权限、索引意图、canonical、语言替代关系和站点地图必须表达同一套页面策略。
- 无法证明页面首选 URL 时，宁可不输出 canonical，也不能默认指向首页。
- 只有真实存在且内容对应的语言版本才能互相声明 hreflang。
- 工具页使用 `noindex`，私有路径继续使用 robots 边界；两者不混作同一种问题。

## 页面策略矩阵

| 页面组                                  | 抓取     | 索引 | canonical       | 语言策略       | sitemap    |
| --------------------------------------- | -------- | ---- | --------------- | -------------- | ---------- |
| 首页、FAQ、镜像介绍、广告落地页、政策页 | 允许     | 是   | 自引用          | 中英双语       | 中英均收录 |
| 中文教程列表、分类、详情                | 允许     | 是   | 中文自引用      | 仅中文         | 仅中文收录 |
| 英文教程列表、分类、详情                | 允许跳转 | 否   | 由 308 跳转合并 | 跳到中文       | 不收录     |
| `/upgrade` 与状态页                     | 允许     | 否   | 不需要          | 不声明替代版本 | 不收录     |
| `/chat` 与子页面                        | 允许     | 否   | 不需要          | 不声明替代版本 | 不收录     |
| 后台、设置、活动、API                   | 禁止     | 否   | 不需要          | 不声明         | 不收录     |

## 元数据

调整通用 `getMetadata`：只有调用方显式传入 `canonicalUrl` 时才生成 canonical 和 Open Graph URL。默认元数据继续提供标题、描述、图片和 robots，但不再把缺少配置的任意页面归到首页。

首页显式传入 `/`。升级操作页、升级状态页和聊天布局显式输出 `noindex, nofollow`。聊天布局当前是客户端组件，因此拆成“服务端元数据布局 + 客户端交互布局”两层，保持 UI 行为不变。

## 语言替代关系

删除根布局手工生成的 `<link rel="alternate">`。真实双语页面继续采用 `next-intl` 代理产生的 HTTP `Link` 响应头。

教程只支持中文：代理在国际化中间件之前识别教程路径和有效语言，将非中文教程以 308 跳到对应中文规范路径，并保留查询字符串。中文教程响应移除 `next-intl` 自动生成的语言替代 `Link` 头，避免声明不存在的英文副本。

该策略集中在代理层，覆盖列表、分类和详情，不依赖数据库是否可用，也不会短暂渲染错误语言正文。未来文章数据模型支持真实多语言后，再移除这条临时语言约束。

## robots 与站点地图

`robots.ts` 删除两条公开政策页禁令，保留设置、活动、后台和 API 禁令。

站点地图由两组记录组成：

1. 真实双语的公开静态页面，按当前默认语言规则生成规范 URL，并附语言替代关系。
2. 仅中文的教程列表和两篇现存详情页，不生成英文副本。

删除 `qf-dtyuedan-buy`、查询页、升级流程、聊天页和已删除文章。静态页面不再填充虚假的当前时间；教程只使用内容 frontmatter 中的稳定发布日期。

## 测试策略

- 扩展现有 `search-console-robots.test.ts`，直接验证 robots 和 sitemap 的 URL 集合、索引资格、语言范围及稳定时间。
- 为 SEO 元数据源码约束、首页/工具页策略和根布局 hreflang 删除增加回归断言。
- 对教程语言路由辅助函数做输入输出测试，覆盖默认语言变化、列表、分类、详情、查询字符串和非教程路径。
- 运行完整测试、ESLint、Prettier 检查和 Next.js 生产构建。

## 风险与回滚

- 308 可能被浏览器和搜索引擎缓存；仅对当前确定没有英文正文的教程启用。回滚时删除教程语言守卫即可。
- 删除错误 canonical 后，未显式配置的页面会暂时没有 canonical，但不会再错误合并到首页；后续可按页面价值逐一补充。
- 站点地图新增政策页会触发重新抓取，但不会改变页面内容或访问权限。
