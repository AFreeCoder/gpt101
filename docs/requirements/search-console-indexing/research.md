## 生产响应与代码对照

1. `/privacy-policy` 与 `/terms-of-service` 的页面元数据允许索引，但 `src/app/robots.ts` 同时禁止抓取，规则互相冲突。
2. `src/app/[locale]/layout.tsx` 对所有页面调用默认元数据；`src/shared/lib/seo.ts` 在没有页面路径时把 canonical 默认为 `/`。因此 `/upgrade`、`/upgrade?source=home` 和 `/chat` 在生产环境都指向首页。
3. `src/app/layout.tsx` 在所有页面硬编码语言替代链接，且把英文指向根路径、中文指向 `/zh`；生产默认语言实际为中文，与国际化路由的规范 URL 相反。
4. `next-intl` 的代理响应已经提供语言替代关系，根布局的第二套规则既重复又冲突。
5. 文章表没有语言字段，远程文章查询也不按语言过滤；仓库内只有两个 `.zh.mdx` 教程。生产 `/en/tutorials/*` 因而在 `lang=en` 页面中输出中文正文。
6. 当前站点地图包含 `noindex` 的 `/qf-dtyuedan-buy`，遗漏 FAQ 和政策页，并对多项静态 URL 使用每次生成都会变化的当前时间。
7. Search Console 的站点地图列表当前没有提交记录；该操作应在代码上线并验证生产 `sitemap.xml` 后执行。
