# Search Console 索引治理需求

状态：已确认
日期：2026-07-15

## 背景

Google Search Console 对 `gpt101.org` 报告了多类页面索引提示。现场复核表明，其中既有真实配置冲突，也有符合预期的排除项和历史 URL。详细证据见 [research.md](./research.md)。

当前主要业务问题是：公开页面可能被阻止抓取或被错误归到首页，站点对不存在的英文教程声明了英文版本，站点地图同时遗漏重要公开页并包含不应索引的工具页。这会让搜索引擎难以判断页面的真实用途和首选 URL。

## 目标

1. 让隐私政策、服务条款等公开内容可被正常抓取和索引。
2. 让可索引页面指向自身的首选 URL，不再默认归到首页。
3. 让聊天、升级操作流程等工具页明确保持非索引状态。
4. 不再把只有中文内容的教程声明成英文页面。
5. 让站点地图只列出真实存在、可索引、首选的公开 URL。
6. 保留后台、设置、活动和 API 等非公开路径的抓取边界。

## 业务规则

- 首页、FAQ、镜像介绍、广告落地页、隐私政策和服务条款属于公开内容，应允许抓取。
- `/upgrade` 是用户完成升级的操作流程，`/chat` 及其子页面包含交互或用户数据；二者不是搜索落地页，应明确排除索引。
- 教程当前只有中文内容。英文教程列表和详情 URL 不应继续呈现中文正文并声称为英文版本；访问者应到达对应的中文首选页面。
- 已有等价新页面的历史教程 URL 继续保留现有跳转；无法证明等价关系的历史 404 不做批量重定向。
- 查询参数不得产生新的站点地图 URL；同一页面的营销来源参数仍归属于该页面自身。

## 验收标准

- `robots.txt` 不再阻止隐私政策和服务条款，同时继续阻止后台、设置、活动和 API 路径。
- 首页和所有明确配置的公开页输出正确的自引用 canonical；缺少页面级配置时不得自动继承首页 canonical。
- `/upgrade`、`/upgrade/status/*`、`/chat`、`/chat/history` 和 `/chat/*` 输出 `noindex`。
- 英文教程列表、分类和详情 URL 永久跳转到对应中文 URL；中文教程响应不声明不存在的英文替代版本。
- 页面 HTML 不再输出错误的全局 hreflang；真实双语页面仍由国际化路由提供语言替代关系。
- 站点地图包含公开规范页和中文教程，排除 noindex、查询参数、英文教程副本及已删除文章。
- 站点地图不得使用每次请求都会变化的“当前时间”伪装页面更新时间。
- 针对上述规则的自动化测试通过，完整测试、静态检查和生产构建不出现新增错误。

## 非目标

- 本次不翻译教程正文，也不为文章表新增多语言字段。
- 本次不处理所有历史 404、错误子域名或外部生成的异常 URL。
- 本次不新增商品结构化数据；当前代码未输出 `Product`，Search Console 中该项为历史状态且最新报告无数据。
- 本次不部署生产环境，也不代替上线后的站点地图提交、验证修复和重新抓取操作。
