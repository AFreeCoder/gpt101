# 生产镜像保留策略开发记录

日期：2026-07-15

关联设计：`docs/design/image-retention-hardening/design.md`

## 实际完成

- 新增 `deploy/image-retention.sh`，支持按不同镜像 ID 保留当前版本和最近成功版本，并支持 `--dry-run`。
- 发布工作流在 PostgreSQL、Web 和 Worker 全部检查通过后执行镜像清理，保留数量固定为三个版本。
- 清理范围限定为 GPT101 回滚标签和当前 GHCR 仓库的本地 `sha-*` 引用。
- 被任何容器引用的旧镜像会被跳过，脚本返回失败，避免强制删除。
- 修正 `deploy/rollback.sh` 的提交号镜像标签，使其与构建流程的 `sha-<完整提交号>` 一致。
- 更新部署事实源和自动化测试。

## 与设计的偏差

无。

## 现场验证

将新脚本通过标准输入发送到生产服务器并执行 `--dry-run`，未写入服务器文件，也未删除镜像。预演结果为：

- 保留当前运行镜像。
- 保留最近两个成功生产镜像。
- 计划清理三个更旧镜像的六个本地引用。
- 没有发现仍被容器引用而无法清理的目标镜像。
