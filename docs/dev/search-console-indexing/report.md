# Search Console 索引治理开发记录

日期：2026-07-15
设计基线：`docs/design/search-console-indexing/design.md`

## 实际完成

- 放开隐私政策和服务条款的抓取权限，保留后台、设置、活动和 API 的 robots 边界。
- 修改通用元数据生成器：调用方未提供页面路径时，不再生成首页 canonical 或首页 Open Graph URL。
- 首页显式配置自引用 canonical；升级流程、升级状态和聊天页面显式配置 `noindex, nofollow`。
- 将聊天布局拆为服务端元数据布局和客户端交互布局，原有侧边栏、上下文和本地化逻辑保持不变。
- 删除根布局中方向相反且作用于所有页面的手工 hreflang。
- 新增纯路径路由工具，在代理层把非中文教程以 308 合并到中文规范 URL，并对教程和 noindex 工具页移除自动语言替代响应头。
- 教程规范跳转固定使用站点配置域名，不信任请求 `Host` 或转发 Host；查询参数继续保留。
- 重建站点地图：收录真实双语公开页和仅中文教程，删除 noindex 页、已删除文章及动态“当前时间”。
- 给镜像介绍页和广告落地页补齐中英文元数据，避免英文页面继续使用中文标题和描述。
- 发布前按真实 Docker 链路复核时发现：镜像构建阶段没有默认语言变量，运行阶段才注入中文；静态生成的 sitemap 会固化构建机英文默认值。因此将 sitemap 改为运行时生成，继续以容器的中文默认语言配置为事实源。

## 实施说明

- 没有修改文章表或生产数据；教程语言策略完全在 URL 边界生效。
- 没有新增历史 URL 重定向。此前已删除且无明确等价关系的文章继续返回 404。
- 为兼容现有升级子域回归测试，保留 `createUpgradeSubdomainRedirectUrl` 入口，由它复用新的公共跳转 URL 构造逻辑；升级子域行为未改变。
- 没有修改 Dockerfile、GitHub Actions 或生产环境变量；仅让依赖运行时语言配置的 sitemap 延后到运行时计算。
- 无数据库迁移、依赖版本或锁文件变更。

## 上线后动作

1. 验证生产环境的 `robots.txt`、`sitemap.xml` 和本文测试报告中的关键 URL。
2. 在 Search Console 提交 `https://gpt101.org/sitemap.xml`。
3. 对“被 robots.txt 阻止”和“已抓取但尚未编入索引”发起验证修复。
4. 分别请求重新抓取首页、FAQ、两项政策页、镜像介绍页、广告落地页和两篇中文教程。
5. 观察至少一个完整抓取周期；预期 `noindex` 和重定向数量仍会存在，但应对应明确的工具页和规范化 URL。
